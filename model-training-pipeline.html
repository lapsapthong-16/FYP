01 / Source

Where the data came from.

I combined domain-specific Reddit crisis data with general benchmark datasets so the models could learn broad sentiment and emotion patterns, then be checked against the Samsung Note 7 crisis domain.

Reddit crisis data

Collected from r/samsung with Reddit API tooling through PRAW. The topic was the Samsung Galaxy Note 7 recall, overheating, fire, refund, replacement, and customer-service crisis.

PRAW r/samsung Note 7

General sentiment data

Used SST-2 from Hugging Face Datasets for sentiment training and validation. It provided a clean benchmark for negative/positive sentiment learning.

SST-2 Hugging Face

General emotion data

Used GoEmotions simplified for emotion classification. Labels were mapped into six project classes: anger, fear, joy, sadness, surprise, and no emotion.

GoEmotions 6 classes
Reddit collection used 11 targeted queries such as "Galaxy Note 7 recall", "Note 7 battery fire", "Note 7 explosion", "Note 7 refund", "Note 7 replacement", and "Note 7 overheating". The repo snapshot contains 329 cleaned rows and 329 annotated rows.
02 / Cleaning

How raw Reddit text became model-ready data.

The goal was to keep the crisis meaning while removing noise that would distract the model. Usernames and formatting were normalized instead of letting the model memorize identifiers.

Collect and flatten

Posts and selected comments were collected, deduplicated, then flattened into JSON/CSV rows.

Files: data-collect.ipynb, flat*.json, combined_y_labeled_data.json

Filter for usable crisis text

English text and subjective crisis discussion were prioritized. Low-subjectivity content was filtered out during collection.

Filters: English check, TextBlob subjectivity threshold around 0.4

Normalize noisy social text

URLs, HTML, Reddit formatting, mentions, extra whitespace, and identifiers were cleaned or anonymized.

Output: data/cleaned_reddit_posts.csv

Add task labels

The annotated data contains both sentiment and emotion labels for the same crisis-domain examples.

Output: annotated_reddit_posts.csv
03 / Split

How I split the data.

The main training notebooks used a reproducible two-stage split. The important part: both split stages used stratification, so each subset kept a similar label balance.

Subset What it was used for Why it mattered
Training Fine-tuned transformer weights. The only split used for gradient updates.
Validation Tracked loss, accuracy, and macro-F1 after epochs. Used for model selection and hyperparameter tuning.
Test Final held-out evaluation. Kept separate from training decisions.
Cross-validation used shuffled KFold, not StratifiedKFold. So the normal train/validation/test split was stratified; the cross-validation folds were shuffled and reproducible, but not explicitly stratified.
04 / Training

How the models were trained.

I did not train language models from zero. I fine-tuned pretrained transformer encoders for the project tasks, then compared single-task and multi-task setups.

BERTweet

Backbone: vinai/bertweet-base. Chosen because crisis discussion is social-media text with usernames, informal grammar, complaints, and short comments.

Social text Transformer

DistilRoBERTa

Backbone: distilroberta-base. Used as a compact general transformer baseline to compare against BERTweet.

Single-task sentiment

One transformer encoder with one classifier head for sentiment prediction.

05 / Tuning

How I tuned the models.

Tuning searched for settings that improved validation macro-F1, because macro-F1 is more honest than accuracy when classes are imbalanced.

Learning rate 2e-5 to 1e-4 searched on log scale in tuning notebooks
Batch size 16 / 32 with smaller batches for heavier multi-task passes
Dropout 0.1 to 0.3 hidden and classifier dropout tuned
Multi-task alpha task weight used to balance sentiment and emotion loss
I tuned on general datasets, then checked both general benchmark performance and Reddit-domain performance. This separates "can the model learn the task?" from "does it transfer to the crisis case study?"
06 / Validation

How I checked the work.

I used several checks because a single score can be misleading: held-out testing, official benchmark validation/test splits, cross-validation, random seeds, t-tests, and explainability metrics.

Held-out test set

The 15% test set was used after training to evaluate unseen samples. Metrics included accuracy and macro-F1.

Official benchmark splits

SST-2 validation and GoEmotions test/validation splits were used in validation notebooks for general-dataset checks.

5-fold cross-validation

DistilRoBERTa cross-validation used KFold with five folds, shuffle enabled, and random_state=42.

Random seed stability

Training was repeated across seeds 42, 123, 456, 789, and 999, then summarized with mean and standard deviation.

Statistical testing

Paired t-tests compared single-task versus multi-task macro-F1 across seeds.

Explainability checks

Captum Integrated Gradients measured token importance, faithfulness, attribution stability, and bias-style ratios.

07 / Results

What the final checks showed.

The clearest result: single-task emotion models were much stronger on general GoEmotions-style validation. On Reddit-domain data, results were noisier and paired tests did not show significant single-task versus multi-task differences.

General-dataset seed stability

BERTweet emotion single-task: macro-F1 0.6395 +/- 0.0985. DistilRoBERTa emotion single-task: macro-F1 0.7066 +/- 0.0046.

Paired t-test summary

General emotion differences between single-task and multi-task were significant. General sentiment was not significant. Reddit-domain comparisons were not significant.

Interpretation for non-technical readers: the pipeline worked best when sentiment and emotion were trained separately. Multi-task learning was useful to test, but it did not consistently improve the final results.
08 / Deployment

How it became usable.

The trained artifacts were designed to be loaded by a Streamlit app for single-text inference, probability display, token attribution, and counterfactual checks.

Model paths

The app expects model folders for sentiment, emotion, and multi-task backbones under a local models directory.

Inference

The app tokenizes the input, runs the selected model, applies softmax, and shows label confidence.

Explainability

It highlights important tokens using Integrated Gradients where available, with a gradient fallback path.

Built from the project notebooks and repository artifacts: data collection, preprocessing, BERTweet, DistilRoBERTa, validation, cross-validation, XAI, and t-test workflows.
